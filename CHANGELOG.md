# Changelog

Development history of this modernized fork, in chronological order. See `README.md` for the current state of the project; this file exists so that history doesn't have to be read to understand how to use it.

- Replaced the Tiny Images dataset with CIFAR-10, Fashion-MNIST, and LFW (facial recognition) in `src/image.py`.
- Removed the TensorFlow 1.x calls that no longer exist in TensorFlow 2.x (`tf.log` -> `tf.math.log`, removed `tf.reset_default_graph()`).
- Updated model construction and optimizers for Keras 3 (explicit `Input` layer, `learning_rate=` keyword).
- Made the TensorFlow import lazy so the Gaussian-field and plotting utilities work without TensorFlow installed.
- Refreshed `requirements.in`/`requirements.txt` for modern Python.
- Added a correctly spelled `LogisticRegression` alias (the original `LogsiticRegression` name still works).
- Fixed `src/mine.py` for the current Keras 3 data-adapter API, which is stricter about generator inputs than the version this project was originally written against:
  - `get_finite_dataset` now yields `(inputs, targets)` as tuples instead of lists — Keras's generator adapter now infers a `tf.TypeSpec` per input and rejects plain lists.
  - `train_steps`/`val_steps` are now cast to `int` (`np.ceil` returns a `numpy.float64`, which the newer epoch iterator no longer accepts in `range()`).
  - Added `mine.cycle_generator`, replacing `itertools.cycle(...)` for repeating the validation generator — Keras's adapter now requires an actual generator object and rejects `itertools.cycle` instances. `examples.ipynb` was updated to match.
- `examples.ipynb`'s "Visualizing the scaling for a real dataset" section now sweeps **all** available real datasets (MNIST, Fashion-MNIST, CIFAR-10, LFW) instead of only MNIST, plotting them together with `image.plot_mi_scaling`.
- That section now plots the *direct* MI estimate rather than the *indirect* (Donsker-Varadhan-style) one, matching how the original paper's own real-dataset figures (`plot_averages`, since removed - see below) were generated. The indirect estimate's `log(mean(exp(...)))` term is much more sensitive to noisy batches and can dip below zero even though MI is analytically non-negative; the direct estimate (a plain mean of classifier logits) is far more stable. `image.plot_mi_scaling` also gained a `clip_negative` option (used here) to floor any residual noise-driven negative values at zero.
- `mine.run_bipartition` gained an optional `eval_steps` argument (default 5000, unchanged) so the number of validation batches averaged over in `evaluate_MI` can be reduced to match a smaller `num_images`, instead of always evaluating over 5000 batches regardless of dataset size.
- The notebook's real-dataset sweep now uses reduced settings (10,000 images, up to 30 epochs, patience 8, 400 eval steps) instead of the paper's full scale (70,000 images, up to 3,000 epochs, see `mine.ini`), so the 27-length sweep across multiple datasets finishes in minutes rather than hours. Raise these values for higher-fidelity curves closer to the published figures.
- Added `lfw_faces` (via `sklearn.datasets.fetch_lfw_people`) as a working facial dataset in `src/image.py`, and included it in the notebook's real-dataset sweep. Added `scikit-learn` to `requirements.in`/`requirements.txt`. This replaced an earlier attempt using the much smaller Olivetti Faces dataset (400 images), whose MI estimates were noisy and squashed flat on the shared-axis comparison plot; LFW's 13,000+ images give comparable-magnitude, much less noisy estimates. The notebook still also plots it alone on its own axis (`image_results/lfw_faces_scaling.pdf`) to check.
- Removed FER-2013 support entirely (`_load_fer2013`/`_load_fer2013_csv` in `src/image.py`, the `fer_csv_path`/`fer_data_dir` arguments to `get_images`, and the `tensorflow-datasets` dependency): FER-2013 has been removed from the `tensorflow-datasets` catalog and can no longer be loaded automatically, and LFW now covers the facial-dataset role instead.
- `examples.ipynb`'s Gaussian Markov random field section gained two new live-computed cells after the static reference figures: (1) the *exact* analytic MI curves for all three field types at both correlation strengths, computed instantly from the closed-form Gaussian formula (`image.get_analytic_MI`) with no training; and (2) a validation plot overlaying the neural estimator's *direct* MI estimate (swept across all 27 partition lengths, same reduced settings as the real-dataset sweep) against that exact curve for the `diffuse`/`large` field, to demonstrate the estimator actually recovers the known value.
- Fixed a notebook-format bug where several cells' `source` field had been written as a single raw string instead of a list of lines (both are valid per the nbformat spec, but the string form rendered as blank/invisible cell input in VS Code's notebook editor even though the cells still executed correctly).
- Added `image.shuffle_pixels_independent` and `image.shuffle_pixels_shared`, plus two new `get_images` sources built on them (`cifar10_shuffle_independent`, `cifar10_shuffle_shared`, since removed - see below), as a real-data analog of the synthetic `sparse` (randomized boundary-law) GMRF test. `examples.ipynb` gained a new section sweeping both across all 27 partition lengths and plotting them against unshuffled CIFAR-10.
- Re-added FER-2013 as `fer2013_hf` (`src/image._load_fer2013_hf`), this time via the `clip-benchmark/wds_fer2013` mirror on the Hugging Face Hub instead of the now-gone `tensorflow-datasets` entry - see README's "A note on FER-2013 (`fer2013_hf`)". Added `datasets` to `requirements.in`/`requirements.txt`.
- `mine.run_bipartition` gained an optional `target_size` argument (default 28, unchanged), passed straight through to `img.get_images`, so a sweep can be run at a dataset's native resolution instead of the default 28x28 crop/resize.
- Added `src/image_noncrop_experiment.py`, a standalone script that reruns the real-dataset MI-scaling sweep for CIFAR-10, LFW, and FER-2013 at each dataset's native (non-cropped) resolution using the new `target_size` argument above, saving results and a combined scaling plot to `image_noncrop_results/` - see README's "Non-cropped MI scaling".
- `src/image_noncrop_experiment.py`'s sweep now runs the entire image (length 1 to `target_size` inclusive) instead of stopping at 27, revealing the full rise-then-fall shape of each curve - see README's "Area law vs. volume law, and what the full-length sweep revealed". Also fixed a latent bug in `plot_combined` where a single shared `lengths` array (from whichever dataset happened to run last) was passed for every series' x-axis - harmless while every dataset swept the same 1-27 range, but silently wrong once each dataset's range differs.
- Added `image_noncrop_experiment.run_sweep`-driven full-range sweeps for MNIST itself (`mnist_mi_*.npy`) and two new `get_images` sources, `mnist_shuffle_independent`/`mnist_shuffle_shared`, as the corrected real-data analog of the synthetic `sparse` GMRF (replacing `cifar10_shuffle_*` in that role) - see README's "A note on `mnist_shuffle_independent`/`mnist_shuffle_shared`".
- Removed `cifar10_shuffle_independent`/`cifar10_shuffle_shared` entirely (the `get_images` sources in `src/image.py`, and the `examples.ipynb` section built on them, including its `cifar10_shuffle_scaling.pdf`/`.png` output) now that `mnist_shuffle_independent`/`mnist_shuffle_shared` fully supersedes them as the real-data analog of the `sparse` GMRF - CIFAR-10 turned out not to satisfy the local-correlation assumption that test depends on (previous bullet). `image.shuffle_pixels_independent`/`shuffle_pixels_shared` themselves are unchanged and are now used by the MNIST sources instead.
- Repository cleanup pass:
  - Split this changelog out of `README.md` into its own file, so the README stays focused on how to use the project rather than its history.
  - Removed six dead functions from `src/image.py` (`plot_cov`, `plot_gaussian_cov_image`, `plot_sampled_images`, the original `plot_gaussian`, `plot_averages`, `plot_large_small_avg`) - leftovers from the original paper author's workflow that depended on large trial-data files (`averages/`, `trials/`) never included in this repo, and had zero remaining callers now that `examples.ipynb` reproduces all of the same figures live via `plot_mi_scaling`/`get_analytic_MI`. Also dropped the now-unused `matplotlib.lines`/`matplotlib.patches` imports those functions needed.
  - Removed `language.shuffle_word_order`, a documented but never-called negative-control helper.
  - Added a `.gitignore` (`__pycache__/`, `*.pyc`, `.venv/`, `.ipynb_checkpoints/`) and stopped tracking `src/__pycache__` in git.
  - Moved the loose root-level result plots (`gaussian_scaling_analytic.*`, `gaussian_scaling_validation.*`, `lfw_faces_scaling.*`, `real_dataset_scaling.*`, `mnist_shuffle_scaling.*`) into a new `image_results/` folder, matching the existing `language_results/`/`image_noncrop_results/` convention. Updated every `save_path=` in `examples.ipynb` and every reference in `README.md` to match.
  - Renamed `mutual_information_non_cropped/` to `image_noncrop_results/` for the same naming consistency, and deleted the raw Keras training logs that had accumulated there (`run.log`, `mnist_shuffle_run.log` - ~9.6 MB combined of per-batch progress-bar output, no archival value).
- Added `CHANGES_FROM_ORIGINAL.md`, a topic-organized (rather than chronological) explanation of every change in this fork relative to a fresh clone of the original [IanConvy/mutual-information-scaling](https://github.com/IanConvy/mutual-information-scaling) repository, verified by an actual file-by-file diff rather than written from memory. Also expanded the code comments in `src/mine.py` and the Gaussian-field math in `src/image.py` - both still carried the original author's fairly terse one-line comments - to explain *why* each piece of the algorithm works the way it does (the classifier-as-density-ratio-estimator trick, the direct/indirect MI formulas, why `get_mixed_indices` needs a duplicate-pairing check, why each GMRF constructor produces area-law vs. volume-law scaling), not just what each line does. No logic was changed; re-verified with the same smoke tests used throughout this project.
